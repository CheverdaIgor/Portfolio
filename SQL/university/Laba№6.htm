Лабораторная работа 6

Администрирование в MS SQL-серверах

Цель: научиться выполнять администрирование баз данных, хранящихся на MS SQL-серверах; экспортировать/импортировать данные; резервировать/восстанавливать базы данных.

Порядок выполнения работы:

  1. Создать нового пользователя Вашей базы данных под именем TEST.
    (Ваша база данных ->Users->контекстное меню ->New User).
  2. Далее создать новые роли в базе данных под именами test_select, test_update, test_deny
    (Ваша база данных ->Roles->контекстное меню ->New Role)    ,
    в которых будут определены права на выборку/обновление и запрещение действий над данными соответственно.
    Приметить созданные роли к новому пользователю.
  3. Попрактиковаться в создании резервной копии Вашей базы данных.
    (Ваша база данных ->контекстное меню ->Все задачи --> Backup Database)
  4. Создать еще одну базу данных на сервере TEYA. Освоить экспортирование данных из ранее созданной Вами базы данных в только что созданную. Для чего необходимо выполнить Ваша база данных ->контекстное меню ->Все задачи --> Export Data, и в диалоговых окнах указать:
    • в качестве источника данных указать: вид источника - Microsoft OLE DB Provider for SQL Server, сервер-источник - TEYA, база данных - Ваша.
    • в качестве приемника данных указать: вид приемника - тот же (Microsoft OLE DB Provider for SQL Server), сервер-приемник - (local), база данных - Ваша новая база данных.
    • указать, что будет происходить копирование таблиц из одной базы данных в другую, и указать, какие именно таблицы нужно скопировать. При необходимости переименовать копируемые таблицы.
    Проверить корректность экспортирования данных.
  5. Создать в Вашей новой базе данных новую таблицу со структурой на Ваше усмотрение. Провести процедуру импортирования этой таблицы из новой базы данных в Вашу старую базу. Для этого выполнить Ваша база данных ->контекстное меню ->Все задачи --> Import Data, и в последующих диалоговых окнах указать:
    • в качестве источника данных указать: вид источника - Microsoft OLE DB Provider for SQL Server, сервер-источник - (local), база данных - Ваша новая база.
    • в качестве приемника данных указать: вид приемника - тот же (Microsoft OLE DB Provider for SQL Server), сервер-приемник - TEYA, база данных - Ваша старая база данных.
    • указать, что будет происходить копирование таблиц из одной базы данных в другую, и указать, какие именно таблицы нужно скопировать. При необходимости переименовать копируемые таблицы.
    Проверить корректность импортирования данных.
  6. Научиться восстанавливать резервную копию сохраненной базы данных.
    (Ваша база данных ->контекстное меню ->Все задачи --> Restore Database)
  7. Оформить отчет по выполненной работе, в котором изложить теоретические сведения, освоенные при выполнении работы, и порядок выполнения работы. Быть готовым к защите лабораторной работы.
